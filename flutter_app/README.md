# 📱 دليل منصة نبرة الذكية للهواتف (تطبيق Android الحقيقي بـ Flutter)

تطبيق **نبرة (Nabrah)** للهواتف الذكية هو تطبيق هاتف أصلي (Native/Hybrid Android App) تم بناؤه باستخدام إطار العمل العالمي **Flutter** ليتكامل كلياً مع منصة الويب والـ Backend الخاصة بمشروعك الناشئ ليكون منتجاً متكاملاً (All-in-One Voice Biotech Startup).

---

## ✨ ميزات تطبيق الهاتف المحمول (Android App Features):
1. **تصميم عصري متناسق (Modern Visual System):** يتبنى هويّة المنصة البصرية الفاخرة (Emerald Premium Theme) مع دعم الخطوط العربية الفاخرة (Cairo & Amiri) باستخدام Google Fonts.
2. **تسجيل صوتي عالي الدقة (Phone Mic Recording):** تقنيات متطورة لالتقاط الصوت من ميكروفون الهاتف مباشرة، مع التحقق التلقائي من الأذونات وتخزين المسارات بصيغة خفيفة ومناسبة للإرسال والتحليل.
3. **مزامنة وتحليل فوري بالذكاء الاصطناعي (AI Analytics Sync):** الاتصال بالخادم المركزي لمعالجة عينات الصوت، تشكيل النصوص آلياً، والعيادة الرقمية التشاركية الفورية.
4. **تخزين وإدارة محلية (Offline-First Local Storage):** لحسابات المستخدمين وسجلات الفحوصات والدرجات المكتسبة باستخدام `SharedPreferences`.
5. **لوحة بيانية تفاعلية (Dynamic Charts):** رسوم بيانية جميلة وتفاعلية تعرض تطور الأداء النطقي الصوتي للمريض عبر الزمن لتعزيز الاستمرارية.

## 📂 الهيكل البرمجي لملفات الهاتف (Flutter Project Layout):

```text
flutter_app/
├── pubspec.yaml                 # المكونات البرمجية والتبعيات ومكتبات الصوت والرسم
├── README.md                    # هذا الدليل التوضيحي المفصل باللغة العربية
└── lib/
    ├── main.dart                # المدخل الرئيسي لبرمجة التطبيق وتهيئة الثيمات العائلية
    ├── services/
    │   └── api_service.dart     # محرك الربط وخادم الويب وإرسال عينات الصوت وملفات الـ Base64
    └── screens/
        ├── login_screen.dart    # شاشة الدخول والتسجيل ورفع الجاهزية الرقمية للمستخدم
        ├── dashboard_screen.dart# اللوحة الرئيسية لمتابعة تطور درجات الصوت ومؤشر التقدم الرسومي
        ├── diacritizer_screen.dart # شاشة التشكيل الآلي وتحويل النصوص إلى كلام (TTS)
        ├── speech_instructor_screen.dart # العيادة التدريبية ومسجل الصوت التفاعلي والتقييم اللفظي
        └── clinical_clinic_screen.dart # عيادة التشخيص المتكامل للأطباء الرقميين والخبير اللساني
```

## 🛠️ متطلبات التهيأة والتشغيل (System Prerequisites):
لتشغيل الكود واستخراج ملف الـ APK، يجب توافر:
- **دعم Flutter SDK:** مثبت على جهازك الإصدار (3.0.0 أو أحدث).
- **محرر أكواد:** VS Code (مع إضافة Flutter extension) أو Android Studio.
- **أدوات تطوير Android:** تثبيت Android SDK Build Tools لصياغة حزمة الجوال.

## 🚀 دليل استخراج ملف الـ APK وتثبيته خطوة بخطوة:

### الخطوة 1: تثبيت التبعيات (Get Packages)
افتح مبوبة تيرمينال في مسار مجلد التطبيق المكتوب بـ Flutter ونفذ الأمر التالي لتحميل كامل المكتبات والصوتيات المطلوبة:
```bash
cd flutter_app
flutter pub get
```

### الخطوة 2: فحص وتأكيد الاتصال بالخادم الرئيسي (API Endpoint Configuration)
سيقوم تطبيق الهاتف بالاتصال بالخادم لمعالجة التحليلات. افتح ملف `lib/services/api_service.dart` وقم بتبديل ثابت الـ Base URL بنطاق خادمك النشط:
```dart
static const String baseUrl = "https://ais-pre-ban7luzvixd2ouzg3tgzz2-896057566989.europe-west2.run.app/api";
```

### الخطوة 3: التحقق والتثبيت على جوالك لتجربته حياً (Debug Mode)
قم بتوصيل هاتفك الـ Android بالكمبيوتر بواسطة كابل USB وتفعيل **خيارات المطورين (USB Debugging)**، ثم نفذ:
```bash
flutter run
```
سيقوم المحرك بربط جوالك وتحميل النسخة الفورية وتجربتها بشكل كامل مع تفعيل تسجيل الصوت والـ API بنجاح.

### الخطوة 4: صياغة ملف الـ APK النهائي للإنتاج (Release Build APK)
لاستخراج نسخة تثبيت نهائية خفيفة ومستقلة تعمل على كافة الأجهزة (Universal APK) بدون الحاجة لجهاز الكمبيوتر، نفذ الأمر التالي:
```bash
flutter build apk --release
```
* لتوليد نسخة تتوافق مع معمارية معينة وتكون فائقة الصغر (Split APKs)، يمكنك تشغيل:
  `flutter build apk --split-per-abi`

### أين تجد ملف الـ APK بعد البناء؟
سيظهر الملف في المسار التالي داخل مجلد مشروعك:
`flutter_app/build/app/outputs/flutter-apk/app-release.apk`
يمكنك نقله فوراً إلى أي جهاز أندرويد بالبلوتوث، الواتساب، أو غوغل درايف وتثبيته مباشرة للتشغيل!

## 🔒 أذونات الأمان المطلوبة لنظام الـ Android:
لقد قمنا بإعداد الأذونات اللازمة تلقائياً في ثنايا الكود، وهي مبرمجة لطلب الإذن فور دخول العيادة:
1. **إذن الميكروفون (`RECORD_AUDIO`):** لتسجيل ونطق المقاطع بدقة.
2. **إذن الإنترنت (`INTERNET`):** للاتصال بخوادم الذكاء الاصطناعي ومعالجة النصوص.
3. **أذونات قراءة المجلدات (`WRITE_EXTERNAL_STORAGE`):** لحفظ المقاطع الصوتية المؤقتة للفحص.
